# Web App Manifest

The package supports two modes for serving the manifest: **static** (default) and **dynamic**.

---

## Static Mode (default)

In static mode the manifest file is generated by Vite / `vite-plugin-pwa` at build time and
placed in `public/build/manifest.webmanifest`. Laravel serves it via the route registered by
the package (or lets Nginx serve the file directly if that file exists on disk).

Switch to static mode (it is the default):

```env
PWA_MANIFEST_DRIVER=static
```

The file `public/build/manifest.webmanifest` is produced automatically when you run
`ddev npm run build` with `laravelPwa()` in `vite.config.js`.

### Fallback before first build

If `public/build/manifest.webmanifest` does not exist yet the `StaticManifestDriver` falls back
to the `pwa.manifest.data` array from `config/pwa.php`. The page renders without error, but the
manifest will contain no hashed asset filenames.

---

## Dynamic Mode

In dynamic mode the `ManifestController` generates the manifest on each request (with optional
caching) using a **resolver** class. This is useful for:

- Multi-tenant apps where each tenant has its own name, icon, and brand colour.
- Localised apps where `name` and `short_name` differ per locale.
- Any scenario where manifest content depends on the authenticated user.

```env
PWA_MANIFEST_DRIVER=dynamic
```

### Default resolver

`DefaultManifestResolver` returns the `pwa.manifest.data` array unchanged. It is a no-op that you
replace with your own class.

### Writing a custom resolver

```php
namespace App\Pwa;

use Illuminate\Http\Request;
use SlFomin\PwaLaravel\Contracts\ManifestResolver;
use SlFomin\PwaLaravel\Manifest\ManifestBuilder;

final class TenantManifestResolver implements ManifestResolver
{
    public function resolve(Request $request, ManifestBuilder $default): ManifestBuilder
    {
        $tenant = $request->user()?->tenant;
        if (! $tenant) {
            return $default;
        }

        return $default
            ->name($tenant->name)
            ->shortName($tenant->short_name ?? $tenant->name)
            ->themeColor($tenant->brand_color ?? '#000000')
            ->startUrl("/t/{$tenant->slug}")
            ->scope("/t/{$tenant->slug}")
            ->icons([
                ['src' => $tenant->icon_192_url, 'sizes' => '192x192', 'type' => 'image/png'],
                ['src' => $tenant->icon_512_url, 'sizes' => '512x512', 'type' => 'image/png'],
            ]);
    }

    public function cacheKey(Request $request): ?string
    {
        // Return null to skip caching for this resolver variant.
        return 'tenant.'.($request->user()?->tenant_id ?? 'guest');
    }
}
```

Bind it in `AppServiceProvider::register()`:

```php
use SlFomin\PwaLaravel\Contracts\ManifestResolver;
use App\Pwa\TenantManifestResolver;

$this->app->bind(ManifestResolver::class, TenantManifestResolver::class);
```

### Caching

Dynamic manifests are cached using Laravel's cache system.

| Config key | Default | Description |
|---|---|---|
| `pwa.manifest.dynamic.cache` | `true` | Enable / disable caching |
| `pwa.manifest.dynamic.cache_ttl` | `3600` | TTL in seconds |
| `pwa.manifest.dynamic.cache_key_prefix` | `pwa.manifest.` | Key prefix |
| `pwa.manifest.dynamic.cache_store` | `null` | Cache store (null = default) |

The cache key is `{prefix}{resolver->cacheKey($request)}`. If `cacheKey()` returns `null` the
manifest is never cached.

### `crossorigin` attribute

The `DynamicManifestDriver` automatically adds `crossorigin="use-credentials"` to the manifest
`<link>` tag. This is required when the manifest URL requires cookies (e.g. session-based auth).

## ManifestBuilder API

`ManifestBuilder` is a fluent DTO that represents a Web App Manifest.

```php
use SlFomin\PwaLaravel\Manifest\ManifestBuilder;

$builder = ManifestBuilder::make()
    ->name('My App')
    ->shortName('App')
    ->description('A great PWA')
    ->startUrl('/')
    ->scope('/')
    ->display('standalone')
    ->themeColor('#1a1a2e')
    ->backgroundColor('#ffffff')
    ->lang('en')
    ->orientation('any')
    ->addIcon('/icons/icon-192x192.png', '192x192', 'image/png', 'any')
    ->addIcon('/icons/icon-512x512.png', '512x512', 'image/png', 'maskable')
    ->shortcuts([
        ['name' => 'Dashboard', 'url' => '/dashboard', 'icons' => [...]],
    ]);

// Outputs validated JSON
$json = $builder->toJson();

// Access raw array
$array = $builder->toArray();
```

### Required fields (validated on serialisation)

- `name`
- `short_name`
- `start_url`
- `display` — must be one of `fullscreen`, `standalone`, `minimal-ui`, `browser`

`ManifestBuilder` implements `JsonSerializable`; passing it to `response()->json()` calls
`validate()` automatically.

## Locale Resolver Example

A locale-based resolver is included as a reference implementation:

```php
use SlFomin\PwaLaravel\Manifest\Resolvers\LocaleManifestResolver;

// In AppServiceProvider:
$this->app->bind(ManifestResolver::class, function () {
    return new LocaleManifestResolver([
        'en' => ['name' => 'My App',       'short_name' => 'App'],
        'fr' => ['name' => 'Mon Appli',    'short_name' => 'Appli'],
        'de' => ['name' => 'Meine Anwendung', 'short_name' => 'Anwendung'],
    ]);
});
```

Cache keys are `locale.en`, `locale.fr`, etc.
